# Module Lifecycle

Every module MUST pass all the following stages before being marked COMPLETE.

- [ ] Product Requirement
- [ ] Architecture Review
- [ ] Backend Implementation
- [ ] Swagger / API Verification
- [ ] Frontend Integration
- [ ] Build Verification
- [ ] Manual Testing
- [ ] Git Commit

Only after all checks pass may a module be marked COMPLETE.

# Architecture Rules

- Reuse existing Repository and Service classes.
- Never change the project architecture.
- One file per step.
- One Git commit after every completed feature.
- Every feature must be tested before being marked COMPLETE.
